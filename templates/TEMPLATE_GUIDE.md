## How to Use

1. **Pick the right template** — use a quickstart if one matches, otherwise start from the master
2. **Replace all [BRACKETED] text** with client-specific details
3. **Adjust pricing** within the standard tiers above (flex ±10% based on scope)
4. **Delete the template usage notes** block at the top before sending
5. **Add client-specific proof points** — always lead with real numbers
6. **Export to .docx** if the client prefers that format

---

## Voice Guidelines (applies to all proposals)

- **Tone:** Founder-to-founder, practical, non-salesy
- **Lead with numbers** — every proposal opens with proof points
- **Frame as investment** — one-time build, they own it, savings compound
- **Always include Calendly CTA** — every proposal ends with a booking link
- **Keep it scannable** — tables, bullet points, bold key numbers
- **Avoid jargon** — write like you're explaining to a smart founder, not a technical buyer
